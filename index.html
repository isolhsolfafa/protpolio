Summary: 실시간 ETL·품질관리로 결측률 30~40% → 0.5% 미만

제조 현장의 QR·작업 로그를 대상으로 실시간 ETL 파이프라인과 데이터 품질 관리 체계를 설계·구현·운영했습니다. 핵심 지표는 아래와 같습니다.

  • 결측률 개선: 30~40% → 0.5% 미만 (60~80배 개선)
  • 리포팅 자동화: 월간 집계 2일 → 실시간 대시보드/슬랙 알림
  • 비용 절감: 유사 외주 대비 99% 이상 절감 (ROI 7,048%)

Python·PostgreSQL·Flask·GitHub Actions·Netlify·Slack API를 결합해 설계→개발→배포→운영까지 End-to-End 단독 수행했으며, 데이터를 신뢰 가능한 상태로 끌어올린 뒤 시각화·알림·AI 예측으로 확장했습니다.

Project: 실시간 ETL 파이프라인 기반 제조 데이터 처리 시스템

산재된 제조 데이터와 높은 결측률(30~40%), 수작업 리포팅 병목을 해결하기 위해, 데이터 품질 관리 선행 → ETL 정규화 → 시각화/알림 자동화 순서로 구축했습니다. 그 결과 결측률을 0.5% 미만으로 낮추고, 리포팅을 실시간화했습니다.

나의 역할: End-to-End 프로젝트 100% 주도

기획, 아키텍처 설계, 백엔드/프론트엔드 개발, 인프라 구축, 배포, 운영 및 고도화까지 프로젝트의 모든 단계를 100% 단독으로 수행하며 아이디어를 완성된 시스템으로 구현했습니다.

  • 기획 및 설계: 현업 요구사항 분석 및 문제 정의, 해결을 위한 시스템 아키텍처 및 데이터 파이프라인 설계
  • 개발: Flask 기반 REST API 서버, PostgreSQL 데이터베이스 모델링, Netlify를 활용한 반응형 웹 대시보드 개발
  • 자동화: Google Sheets/Drive 연동을 통한 수작업 자동화, Slack/카카오톡 기반의 알림 및 챗봇 기능 구현
  • 운영 및 배포: 사내 VM 환경 구성 및 시스템 배포, JWT 기반 보안 적용, 지속적인 사용자 피드백 기반 기능 개선

주요 기능 및 아키텍처

최소한의 비용으로 최대의 효율을 내기 위해 경량의 최신 기술 스택을 조합하여 유연하고 확장 가능한 구조로 설계되었습니다.

  • 실시간 대시보드: 공정 진행률, 불량률, CT(Cycle Time), 히트맵 등 핵심 KPI를 실시간으로 시각화하여 빠른 의사결정 지원
  • 업무 자동화: Google Drive 내 파일 검색 및 하이퍼링크 자동 생성, 메타 데이터 자동 적재를 통해 100% 수작업 제거
  • 안정적인 데이터 파이프라인: JSON 데이터를 정규화하여 PostgreSQL에 자동 적재, 데이터 신뢰성 확보 및 분석 기반 마련
  • 지능형 알림 시스템: 이상 상황 발생 시 Slack, 카카오톡으로 즉시 알림을 전송하여 대응 시간 최소화

핵심 성과: 데이터 품질 혁신을 통한 현장 의사결정 신뢰성 확보

🎯 데이터 위생(Data Hygiene) 체계 구축

QR·현장 데이터 결측률을 30~40% → 0.5% 미만으로 낮추는 데이터 품질 관리 체계를 구축. 이를 기반으로 CT IQR(중앙 50%) 분석을 표준화하고 차이율·신뢰도 등급·샘플 커버리지를 FE/슬랙봇에 일관 제공하여 현장 의사결정 신뢰성과 속도를 혁신적으로 향상시켰습니다.

  • 데이터 결측률 개선: 30~40% → 0.5% 미만 (60~80배 개선)
  • CT IQR 분석 신뢰화: 중앙 50% 분석이 '신뢰 가능한 지표'로 정착
  • 현장 의사결정 체계: 차이율·신뢰도 등급·샘플 커버리지 기반의 표준화된 분석 제공
  • 데이터 선순환 구조: 데이터 위생 → 신뢰도 확보 → IQR 분석 가능화 → 현장 의사결정

💰 재무적 성과: ROI 7,048% 달성

데이터 품질 혁신과 더불어 명확한 재무적 임팩트도 창출했습니다.

  • 총 투자비: ₩2,000,000 (사내 인프라 구성 비용)
  • 총 순이익: ₩140,960,000 (외주 개발비 절감 + 운영 효율화)
  • ROI (투자수익률): 7,048%
  • Pay-back (투자 회수 기간): 약 5일

약 3.5억 원이 소요되는 유사 외주 시스템과 비교했을 때, 99% 이상의 비용을 절감하면서도 데이터 품질 관리, Slack 챗봇, ML 예측 연계 등 더 유연하고 확장성 높은 기능을 확보한 결과입니다.

*모든 성과 수치는 'TwinfaX2 포트폴리오' 문서에 근거합니다. (2025-07-25 기준)

Staging & 3‑Tier 업그레이드 (v3.4.0)

모듈 의존성 정리 및 신규 utils 모듈 반영, 3‑Tier 환경 도입으로 안정성·확장성을 크게 개선했습니다.

  • BE‑staging 환경 구축: GitHub Actions 기반 완전한 테스트 환경, TwinfaX2/BE-staging 저장소로 독립 운영
  • 1‑cycle 테스트: 로컬과 동일하게 추출 → DB 적재 전체 사이클을 CI 상에서 검증
  • 보안/설정: GitHub Secrets로 환경변수 안전 관리
  • Zero‑risk 배포: Railway 배포 전 사전 검증으로 운영 리스크 제거
  • 데이터 소스 전환: 스프레드시트 하이퍼링크 접근 → Google Drive API 직접 검색 및 날짜 필터링
  • 성능/안정성: API 호출 최적화 + 배치 처리로 처리량·안정성 향상
  • 확장성: 시트 구조 변경에 독립적인 설계(robust), 모듈 간 의존성 최소화

Case Studies

Case 1 — 실시간 ETL/품질관리

  • 문제(S): QR/현장 로그 결측률 30~40%, 월간 보고서 2일 소요, 데이터 신뢰도 부족
  • 과제(T): 데이터 신뢰성 확보 후 실시간화
  • 행동(A): 품질 규칙(누락·형식·범위) 설계 → 정규화 스키마/CTE 적용 → 배치 파이프라인(GitHub Actions) → 시각화·슬랙 알림
  • 결과(R): 결측률 0.5% 미만, 리포팅 실시간화, ROI 7,048%
    • 작업시간 누락률 감소 → BE‑FE 구조(Netlify 대시보드)에서 모델/공정별 progress 실시간 노출
    • CT 확인 체계 정립 → 작업시간 IQR 사분위(중앙 50%) 기반 평균 CT 산출·모니터링 가능

Case 2 — AI 예측 & FE 연동

  • 문제(S): 불량 사전 인지 미흡, 우선 대응 품목 선정 지연
  • 과제(T): 텍스트 기반 불량 패턴 조기 탐지
  • 행동(A): Mecab 토크나이저 → TF‑IDF → RandomForest 베이스라인, 상위 특성/키워드 카드뷰, 라벨·샘플링 전략 정립
  • 결과(R): 우선 대응 품목 선정 리드타임 단축(데모 기준), Next Steps: LightGBM 재학습·AUC ≥ 0.85 목표

Case 3 — 협력사 KPI 대시보드

  • 문제(S): 월말 수작업 평가(약 2일), 기준 불일치, 산출 근거 추적 어려움
  • 과제(T): 평가지수(=KPI) 표준화·자동 계산과 추적성(재현성) 확보
  • 행동(A):
    • CI/CD: GitHub Actions로 PDA-defect, nan-dashboard 워크플로우 실행 → 산출물(JSON) 자동 생성·배포
    • 데이터 저장: SQL 적재 없이 JSON 아티팩트를 GitHub에 버전 관리(근거 보존)
    • 산식 적용: 하드코딩된 규칙/알고리즘으로 평가지수 계산(모델·협력사별)
    • 출력/접근제어: Netlify FE에서 지표 시각화, Netlify Identity로 접근 제한
  • 결과(R): 평가 리드타임 2일 → 실시간, 기준 단일화·버전 추적, 협력사 간 비교 재현성 강화

Case 4 — Mobile App Integration (진행 중, 2025.09~)

  • 문제(S): 수기 Google Sheets 입력으로 인한 데이터 지연, 실시간성 부족
  • 과제(T): QR 스캔 기반 즉시 작업 기록 및 실시간 데이터 전송
  • 행동(A):
    • Flutter APP 개발: QR 스캔 → 작업 시작/완료 즉시 기록
    • Connection Pool 최적화: DB 연결 관리로 응답시간 97% 개선 (15-45s → <500ms)
    • Hybrid 운영 전략: 기존 Google Sheets 시스템 유지 + APP 병렬 운영
    • 오프라인 지원: SQLite 로컬 저장 + 네트워크 복구 시 자동 동기화
    • 0.1ms 캐시 조회: QR 메타데이터 초고속 조회 시스템 구축
  • 현재 진행률: Phase 1 완료 - Flask API 4개 엔드포인트 구현, 오프라인 동기화 시스템, 통합 테스트 15개 케이스 검증
  • 다음 단계: WebSocket 실시간 동기화, Dashboard 통합 (Phase 2)
