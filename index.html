제조 DX 5단계 진화 — 현재 위치

G-AXIS의 최종 목표는 대시보드 구축이 아닌 Dark Factory 수준의 자율 제조 운영 체계 구현.
시스템은 5단계로 진화하며, 현재는 운영시스템 안정화를 마치고 분석시스템에 진입한 단계.

6월 핵심 미션

화면을 더 만드는 단계가 아니라, APS·Dark Factory로 확장하기 위한 데이터 신뢰성 기반을 만드는 단계. APS와 Dark Factory는 결국 CT·M/H·리드타임·작업상태 데이터를 믿을 수 있어야 가능하다. 지금의 M/H 계산 정본화·CT 신뢰도 기준·계산 버전 관리가 미래 자율 제조 운영의 기초 체력.

무형 데이터 → 유형 Data Thread

G-AXIS의 본질적 가치는 단순히 시스템을 만든 것이 아닙니다.

Case Studies

프로젝트 성과

01
PDA 제조 DX 통합 시스템 (2025.02 ~)

6-Repo Multi-Service Hybrid 아키텍처로 단독 설계·1년 운영. Flask · PostgreSQL · Slack GPT 챗봇 · Kakao 알림 통합. RandomForest+TF-IDF 불량 예측 등 6개 모듈 운영. 결측률 30%→1% 미만, 외주 1억 대비 내부 200만원 구축으로 ROI 7,048% 달성. 2025년 전사 우수사원·혁신과제 2등 수상.

02
Pull → App Push 아키텍처 전환 결단 (2026.02 ~)

PDA 1년 운영 중 Pull 기반 ETL의 구조적 한계 (M/M 실측 불가·임의 입력 가능) 인지 → 분석 보완이 아닌 데이터 수집 매개체 자체를 재설계 결단. PDA 1년 자산을 기반으로 4개월에 G-AXIS로 진화. 단순 시스템 개발이 아닌 제조 DX 진화 단계 설계 경험. 외부 SI 견적 6.5억원 규모를 단독 완성.

03
G-AXIS 3-Product 구조 운영

OPS (Flutter PWA 현장앱) + VIEW (React 19 + TypeScript 대시보드 21 페이지) + CORE (PostgreSQL 데이터허브 8 스키마·35 테이블). 약 200명 24/7 실사용 환경 안착. 협력사 작업자 현장 교육 완료(MECH·ELEC 2회차)·체크리스트 기반 공정완료(MECH 73항목·ELEC 31항목)·QR 태스크 운영. JWT 자체 인증(access+refresh·PIN·이메일)·Railway/Netlify CI/CD·1,700+ 자동 테스트 기반 운영 안정성.

04
분석시스템 진입 — 6/2 전환점 (2026.06 ~)

Stage 1 안정화 직후 같은 날 두 사건이 교차. 운영 정착(협력사 작업자 현장 교육 완료, 동의서 수집 진행)과 분석 진입(interval-union 계산 fix 적용). 단순 화면 개발이 아닌 DB 설계·계산 정본화·지표 신뢰성 확보 단계로 전환.

5가지 축: elapsed ≠ duration 분리, interval-union 기반 M/H 계산, CT 품질 필터(평균이 아닌 표준공수 적격성 판정), 종료누락 분석(계산 오염 방지), calculation_version 관리(과거·현재 계산 기준 추적). APS·Dark Factory 확장의 선결 조건.

6/2
전환점
운영 정착 협력사 작업자 현장 교육 완료 — 라이브 매뉴얼 시연으로 QR→시작→완료, 멀티작업자, 휴게 자동 pause, 체크리스트 정착.
분석 진입 같은 날 interval-union 계산 fix 적용 — 이후 데이터(completed_at ≥ 2026-06-02)부터 새 로직.
05
AI-Native Harness Engineering (메타 자산)

LLM 협업을 위한 Production-grade 거버넌스 시스템 자체 설계. 6차원 정보 매트릭스 + 3주체 교차검증 + 9단계 state machine + 정량 안전장치. Andrej Karpathy의 LLM coding 4원칙의 superset으로 자체 검증 완료. Production-grade Level 4 단일 프로젝트 하네스. PDA·G-AXIS 1인 구축 효율의 메타 자산이자, "AI 도구 사용자" 가 아닌 "AI 협업 운영 시스템 설계자" 의 결정적 산출물.

06
Cross-Repo 동기화 0~5일 (산업 5-10x)

OPS Sprint 80+ · VIEW Sprint 41+ 동시 운영 중 BE-FE 동기화 0~5일 사이클 달성. 11개 사례 중 5건은 0일 동기화 (같은 commit 단위 작업). 일반 기업 BE-FE 동기화 1주~1개월 대비 5-10배 빠름. 같은 사람이 양쪽 작업 → 컨텍스트 전환 비용 0. ADR 42건 누적으로 Bus Factor 완화.

07
브랜드 & 디자인 시스템

G-AXIS 네이밍부터 로고, GxColors 컬러 토큰, DM Sans + JetBrains Mono 타이포, 컴포넌트 가이드까지 직접 설계. Flutter + React 전체에 일관된 디자인 시스템 적용.
